Part 1 · The Mental Model
The only model you need first

What is a machine actually learning?

Before any algorithm, any code, any equation — you need this one picture in your head. Every supervised learning system in existence is doing exactly one thing:

Find a function f() such that f(input) ≈ correct output, for every example you've seen — and hopefully for examples you've never seen too.

That's it. The "function" is your model. The "correct outputs" are your labels. The process of finding f() is called training.

Imagine you're teaching a new employee to quote prices for custom furniture orders. You show them 500 past orders — sofa dimensions, wood type, cushion material, and the price charged. After enough examples, they start seeing patterns: "bigger sofas cost more, oak is pricier than pine." Now they can quote new orders they've never seen. That employee is a supervised ML model. The 500 past orders are your training data.

The critical word in all of this is "generalise." A model that only memorises your training examples is useless. The whole game is learning the underlying pattern well enough to handle new, unseen data correctly.

Part 2 · Training Fundamentals
Before any algorithm

The four things every training run needs

1 · The Data Split — Never Cheat on Your Exam

You never test a student on questions they've already memorised. Same rule here. Split all your data into three buckets before you touch any algorithm:

The right way — 3 splits
  • Train (70%) — model learns here
  • Validation (15%) — you tune here
  • Test (15%) — final exam, touch once
The wrong way — 2 splits
  • Train + tune on same data
  • Test on "held out" data only
  • Looks good, fails in production

The validation set is your mock exam — you see the results and adjust your approach. The test set is the real exam — you see it once, at the very end, and whatever score you get is your honest model performance.

2 · Loss Function — Your Single Compass

The loss function is one number that answers: "How wrong is my model right now?" Low loss = good model. During training, everything you do has one goal: make this number go down.

Key insight

The choice of loss function is one of the most important design decisions in ML. For regression (predicting numbers), we use MSE. For classification (predicting categories), we use cross-entropy. Different problems, different compasses.

3 · Gradient Descent — Downhill, Eyes Closed

Gradient descent is the algorithm that adjusts your model's knobs (weights) to reduce the loss. Imagine you're blindfolded on a hilly terrain and want to reach the lowest valley. You can't see, but you can feel the slope under your feet. Step downhill. Repeat. That's gradient descent.

It comes in three flavours, each with a different trade-off:

VariantData used per updateSpeedStabilityUsed in practice
BGDAll training dataSlowVery stableRarely
SGD1 random exampleVery fastNoisySometimes
Mini-batch32–128 examplesFastBalancedAlmost always

Mini-batch is what trains GPT-4, trains your house price model, and trains every serious ML model in between. When someone says "batch size 64", they mean mini-batch gradient descent with 64 examples per update.

4 · Overfitting vs Underfitting — The Eternal Enemy

This is the central problem of all of machine learning. Every technique you'll learn — regularisation, cross-validation, dropout — exists to fight it.

Underfitting
Train loss: HIGH
Test loss: HIGH
Model too simple. Barely opened the book.
Just Right ✓
Train loss: LOW
Test loss: LOW
Learned the pattern, not the noise.
Overfitting
Train loss: VERY LOW
Test loss: HIGH
Memorised answers. Can't handle new questions.
Bias-Variance Tradeoff

Underfitting = high bias (model makes wrong assumptions consistently). Overfitting = high variance (model is too sensitive to training data). You can't eliminate both simultaneously — reducing one tends to increase the other. Your job is finding the sweet spot.

Part 3 · Regression
When the answer is a number

Regression — teaching machines to predict quantities

Regression = your model's output is a continuous number. House price. Temperature tomorrow. A user's session time. A patient's blood pressure. Any time the answer is "how much" or "how many" — that's a regression problem.

Linear Regression
Regression · Foundation
Fits the best straight line through your data. The entire model is: y = w × x + b. Two parameters — slope (w) and intercept (b). Learned by gradient descent.
Use when: You suspect a roughly straight-line relationship. Always your baseline model.
Polynomial Regression
Regression · Curved data
Extends linear regression by adding x², x³... terms. Allows curves. Degree 1 = straight line. Degree 3 = gentle S-curve. Degree 15 = overfitting disaster.
Use when: Your data clearly curves and linear regression leaves too much error. Keep degree low (2–4).
Ridge Regression
Regression · L2 Regularisation
Linear regression + a penalty on large weights. Forces the model to keep weights small, preventing it from obsessing over noise. Weights shrink toward zero but never reach it.
Use when: Many features, possible redundancy. All features probably matter a little.
Lasso Regression
Regression · L1 Regularisation
Like Ridge, but the penalty can push weights all the way to exactly zero — automatically removing useless features. Built-in feature selection.
Use when: Many features and you suspect most are irrelevant. Let Lasso decide which ones matter.
Ridge vs Lasso — the 10-second rule

If all features probably contribute something → Ridge (shrinks but keeps all). If most features are probably noise → Lasso (zeroes out the useless ones). When in doubt, try both and compare validation scores.

Regression metrics — how do you measure "how wrong"?

MetricWhat it measuresWhen to use it
MAE Average absolute error. "On average, off by X rupees." Easy to understand, not sensitive to outliers. When you need to explain performance to non-technical stakeholders.
RMSE Root Mean Squared Error. Same units as output. Punishes big errors harder than MAE. Most common for comparing models. The standard reported metric.
MSE Mean Squared Error. RMSE without the square root. Not in original units. During training (smooth gradients), not for reporting to humans.
R² % of variance explained by your model. 0.85 = model explains 85% of variation in output. Expressing overall model quality. 1.0 = perfect, 0.0 = no better than guessing the mean.
Part 4 · Classification
When the answer is a category

Classification — teaching machines to make decisions

Classification = your model's output is a category. Spam or not spam. Disease or healthy. Dog, cat, or rabbit. Fraud or legitimate transaction. Whenever the answer is a label from a fixed set — that's classification.

Logistic Regression
Classification · Probabilistic
Despite the name, it's a classifier. Takes a linear combination of features and squashes it through a sigmoid function to output a probability between 0 and 1. Decision boundary: if P > 0.5, predict class 1.
Use when: Binary classification, you need a probability score, fast training speed matters. Always your baseline classifier.
Decision Tree
Classification · Interpretable
Makes a sequence of yes/no questions (splits) on features to arrive at a prediction. Like a flowchart. Fully interpretable — you can see exactly why it made a decision.
Use when: You need to explain predictions to humans (medical, legal, finance). Beware: deep trees overfit aggressively.
Random Forest
Classification · Ensemble
Build 100+ decision trees, each trained on a random subset of data and features. For each prediction, every tree votes. The majority wins. The randomness prevents any single tree from overfitting.
Use when: You want strong accuracy without much tuning. One of the most reliable algorithms in existence. Almost always better than a single tree.
K-Nearest Neighbours
Classification · Distance-based
To classify a new point, find the K most similar training examples (neighbours) and take a majority vote of their labels. No training phase — it just stores all data and computes at prediction time.
Use when: Small dataset, you need a quick baseline, local patterns matter. Slow at prediction time on large data. Always scale features first.
Naive Bayes
Classification · Probabilistic
Uses Bayes' theorem to compute P(class | features). "Naive" because it assumes all features are independent of each other — a strong assumption that's rarely true but works surprisingly well in practice.
Use when: Text classification (spam detection, sentiment). Extremely fast, works well on small datasets, handles high-dimensional data gracefully.
SVM
Classification · Geometric
Finds the hyperplane (line in 2D, plane in 3D, hyperplane in nD) that maximises the margin between classes. The points closest to the boundary are called support vectors. The kernel trick lets it handle non-linear data.
Use when: High-dimensional data, text classification, clear margin of separation exists. Doesn't scale well to millions of examples.

Classification metrics — accuracy is almost never enough

Here's a mistake every beginner makes: they train a classifier, see "95% accuracy" and celebrate. Then they realise 95% of their data was class 0, so the model just predicts 0 every time and achieves 95% accuracy while being completely useless. Here's how to measure correctly:

MetricWhat it measuresWhen it matters
Accuracy % of all predictions that are correct. Simple, intuitive. Only reliable when classes are balanced (roughly equal number of each class).
Precision Of everything I predicted as positive, how many actually were? Measures false alarms. When false positives are costly. E.g. wrongly flagging a legitimate transaction as fraud.
Recall Of all actual positives, how many did I catch? Measures misses. When false negatives are costly. E.g. missing a cancer diagnosis.
F1 Score Harmonic mean of precision and recall. Single balanced metric. When you need one number and classes are imbalanced. Most commonly used in practice.
Confusion Matrix 2×2 grid: true positives, false positives, true negatives, false negatives. Always look at this. It tells you what kind of mistakes your model makes.
ROC-AUC Area under the ROC curve. Measures how well the model separates classes at all thresholds. Comparing models overall. 0.5 = random guessing. 1.0 = perfect.
The imbalanced classes trap

If 99% of your data is "not fraud" and 1% is "fraud" — a model predicting "not fraud" every single time achieves 99% accuracy. Never report only accuracy on imbalanced datasets. Always use F1 or ROC-AUC.

Part 5 · Regularisation & Cross-Validation
Fighting overfitting

Regularisation — forcing your model to stay humble

Regularisation means adding a penalty to your loss function that punishes the model for being too complex. It's like telling a student: "You'll lose 5 marks for every line of working you write. Be concise."

Ridge Loss = MSE + λ × (w₁² + w₂² + ... + wₙ²)
Lasso Loss = MSE + λ × (|w₁| + |w₂| + ... + |wₙ|)

The λ (lambda) is your regularisation strength hyperparameter. Higher λ = stronger penalty = simpler model = less overfitting (but risk of underfitting). It's a dial you tune during validation.

Ridge (L2)
  • Shrinks all weights toward zero
  • No weight ever becomes exactly zero
  • Keeps all features, reduces their influence
  • Good when all features matter
Lasso (L1)
  • Shrinks weights AND can zero them out
  • Some weights become exactly zero
  • Automatic feature selection
  • Good when most features are noise
Reliable model evaluation

k-Fold Cross Validation — testing your model fairly

With a small dataset, your validation set might be a lucky or unlucky sample. k-Fold solves this by rotating the validation set across your entire dataset:

How 5-Fold Cross Validation works

Split data into 5 equal folds. Round 1: train on folds 2–5, validate on fold 1. Round 2: train on 1,3–5, validate on fold 2. Continue until every fold has been the validation set exactly once. Final score = average of all 5 rounds. Every data point gets tested exactly once. Far more reliable than a single split.

Cross-validation is how you tune hyperparameters (like λ in Ridge/Lasso, or number of trees in Random Forest). For each hyperparameter value, run 5-fold CV and record the average score. Pick the value with the best average. Your test set remains completely untouched throughout all of this.

Cell 1 — Setup & Data

import numpy as np                 # numpy: fast math on arrays
import pandas as pd               # pandas: tables of data (DataFrames)
import matplotlib.pyplot as plt   # matplotlib: draw charts

# StandardScaler: standardises features to mean=0, std=1

    mean_squared_error,          # MSE metric
    r2_score,                    # R² metric
    accuracy_score,              # classification accuracy
    classification_report,       # precision, recall, F1 all at once
    confusion_matrix             # the 2×2 truth table
)
import warnings

np.random.seed(42)  # set random seed → same results every run (reproducibility)

Cell 2 — Regression: Linear, Ridge, Lasso
from sklearn.linear_model import LinearRegression, Ridge, Lasso
# importing three regression algorithms from sklearn's linear_model module

# ── Create a regression dataset ───────────────────────────────
X_reg, y_reg = make_regression(
    n_samples=200,     # generate 200 data points (rows)
    n_features=10,     # each point has 10 features (columns)
    n_informative=5,   # but only 5 of those features actually predict y
    noise=20,           # add some randomness/noise to make it realistic
    random_state=42     # same seed as before → reproducible
)

# ── Split into train and test ──────────────────────────────────
X_train, X_test, y_train, y_test = train_test_split(
    X_reg, y_reg,       # the features and labels to split
    test_size=0.2,      # 20% of data goes to test (80% to train)
    random_state=42     # reproducible split
)

# ── Scale features ─────────────────────────────────────────────
scaler = StandardScaler()          # create the scaler object
X_train = scaler.fit_transform(X_train)
# fit_transform: learn mean/std FROM train data, then scale train data
X_test = scaler.transform(X_test)
# transform only (NOT fit): use the same mean/std from training to scale test
# IMPORTANT: never fit the scaler on test data — that would be data leakage

# ── Train all three models ─────────────────────────────────────
models_reg = {
    'Linear Regression': LinearRegression(),      # plain linear model
    'Ridge (L2)':        Ridge(alpha=1.0),          # alpha = λ (regularisation strength)
    'Lasso (L1)':        Lasso(alpha=0.1)           # lower alpha here since Lasso is more aggressive
}

print(f"{'Model':25} {'MAE':>8} {'RMSE':>8} {'R²':>8}")
print("-" * 53)                              # print a visual separator line

for name, model in models_reg.items():    # loop through each model
    model.fit(X_train, y_train)             # fit = train the model on training data
    y_pred = model.predict(X_test)         # predict outputs for test features

    mae  = mean_absolute_error(y_test, y_pred)             # compute MAE
    rmse = np.sqrt(mean_squared_error(y_test, y_pred))      # compute RMSE (sqrt of MSE)
    r2   = r2_score(y_test, y_pred)                        # compute R²

    print(f"{name:25} {mae:>8.2f} {rmse:>8.2f} {r2:>8.3f}")
# :.2f = print with 2 decimal places, >8 = right-align in 8-character column

# ── Show which features Lasso zeroed out ──────────────────────
lasso = models_reg['Lasso (L1)']           # get our trained Lasso model
zeroed = np.sum(lasso.coef_ == 0)          # count weights that are exactly zero
print(f"\nLasso zeroed out {zeroed} of 10 features (automatic feature selection!)")
Cell 3 — Classification: Logistic, Tree, Random Forest, KNN, Naive Bayes, SVM
from sklearn.linear_model  import LogisticRegression
from sklearn.tree          import DecisionTreeClassifier
from sklearn.ensemble      import RandomForestClassifier
# ensemble = combine multiple models (forest of trees)
from sklearn.neighbors     import KNeighborsClassifier
# neighbors = KNN looks at nearest neighbors
from sklearn.naive_bayes   import GaussianNB
# GaussianNB = Naive Bayes for continuous features
from sklearn.svm           import SVC
# SVC = Support Vector Classifier

# ── Create a classification dataset ───────────────────────────
X_cls, y_cls = make_classification(
    n_samples=500,       # 500 data points
    n_features=10,       # 10 input features
    n_informative=6,     # 6 actually matter
    n_redundant=2,       # 2 are just combinations of informative ones
    n_classes=2,         # binary: class 0 or class 1
    random_state=42
)

# ── Split and scale ────────────────────────────────────────────
X_tr, X_te, y_tr, y_te = train_test_split(
    X_cls, y_cls, test_size=0.2, random_state=42
)
sc = StandardScaler()
X_tr = sc.fit_transform(X_tr)   # learn scale from train, apply to train
X_te = sc.transform(X_te)       # apply same scale to test (no fitting!)

# ── Define all six classifiers ─────────────────────────────────
classifiers = {
    'Logistic Regression': LogisticRegression(max_iter=1000),
    # max_iter: maximum gradient descent steps before giving up

    'Decision Tree':       DecisionTreeClassifier(max_depth=5),
    # max_depth=5: tree can ask at most 5 questions (prevents overfitting)

    'Random Forest':       RandomForestClassifier(n_estimators=100, random_state=42),
    # n_estimators=100: build 100 trees and let them vote

    'KNN':                 KNeighborsClassifier(n_neighbors=5),
    # n_neighbors=5: look at 5 nearest neighbours and take majority vote

    'Naive Bayes':         GaussianNB(),
    # GaussianNB: assumes features follow a Gaussian (bell curve) distribution

    'SVM':                 SVC(kernel='rbf', probability=True),
    # kernel='rbf': radial basis function — handles non-linear data
    # probability=True: allows SVC to output probabilities, not just labels
}

# ── Train and evaluate all classifiers ────────────────────────
print(f"{'Model':25} {'Accuracy':>10} {'F1 Score':>10}")
print("-" * 48)

from sklearn.metrics import f1_score  # F1 = balanced precision + recall

for name, clf in classifiers.items():
    clf.fit(X_tr, y_tr)                          # train on training data
    y_pred = clf.predict(X_te)                  # predict on test features
    acc = accuracy_score(y_te, y_pred)           # what % were correct?
    f1  = f1_score(y_te, y_pred)                # balanced precision-recall metric
    print(f"{name:25} {acc:>10.3f} {f1:>10.3f}")

Cell 5 — k-Fold Cross Validation
# ── Compare all classifiers using 5-fold cross validation ─────
print("=== 5-Fold Cross Validation Scores ===\n")
print(f"{'Model':25} {'Mean CV Score':>15} {'Std Dev':>10}")
print("-" * 53)

for name, clf_obj in classifiers.items():
    scores = cross_val_score(
        clf_obj,          # the classifier to evaluate
        X_cls,            # ALL data (not just train) — CV handles splitting internally
        y_cls,            # all labels
        cv=5,             # 5 folds — split into 5, rotate validation set
        scoring='f1'      # use F1 score as our metric for each fold
    )
    # scores is now an array of 5 numbers — one score per fold
    print(f"{name:25} {scores.mean():>15.3f} {scores.std():>10.3f}")
    # mean() = average performance across 5 folds
    # std() = how consistent the model is (low std = reliable, high std = unstable)

print("\n💡 Tip: low std dev = model is consistent across different data slices")
Cell 6 — Visualise Overfitting vs Underfitting
from sklearn.preprocessing import PolynomialFeatures
# PolynomialFeatures: creates x², x³... features from existing features
from sklearn.pipeline import make_pipeline
# make_pipeline: chain steps (scale → transform → model) into one clean object

# ── Create a 1D curved dataset to visualise clearly ───────────
X_1d = np.linspace(0, 10, 30).reshape(-1, 1)
# linspace: 30 evenly spaced numbers from 0 to 10
# reshape(-1, 1): convert from 1D array to 2D column (sklearn needs 2D input)
y_1d = (2 * X_1d.ravel()
        + np.sin(X_1d.ravel()) * 3
        + np.random.normal(0, 1.5, 30))
# ravel(): flatten from 2D back to 1D for the math operations
# np.sin(): adds a gentle wave (curve) to the data
# np.random.normal(): adds noise — mean 0, std 1.5, 30 values

X_smooth = np.linspace(0, 10, 300).reshape(-1, 1)
# 300 points for a smooth prediction line (more points = smoother curve)

fig, axes = plt.subplots(1, 3, figsize=(15, 5))
# subplots(1, 3): one row, three columns of charts side by side

configs = [
    (1,  'Underfitting\n(degree 1)',  '#c0002a'),  # straight line on curved data = underfit
    (3,  'Just Right\n(degree 3)',   '#0a7c4a'),  # gentle curve = good fit
    (15, 'Overfitting\n(degree 15)', '#a05c00'),  # wild curve = memorised noise
]

for ax, (degree, title, color) in zip(axes, configs):
    # zip pairs each subplot (ax) with its config (degree, title, color)

    model = make_pipeline(
        PolynomialFeatures(degree),  # step 1: create x^degree features
        LinearRegression()           # step 2: fit linear regression on those features
    )
    model.fit(X_1d, y_1d)            # train on the noisy data
    y_line = model.predict(X_smooth) # predict on the smooth line for plotting

    ax.scatter(X_1d, y_1d, color='gray', s=40, alpha=0.7)  # draw data points
    ax.plot(X_smooth, y_line, color=color, linewidth=2.5)     # draw model curve
    ax.set_title(title, fontsize=12, fontweight='bold')
    ax.set_ylim(-5, 30)    # fix y-axis range so all 3 charts are comparable
    ax.grid(True, alpha=0.2)

plt.suptitle('Underfitting vs Just Right vs Overfitting', fontsize=14, y=1.02)
plt.tight_layout()
plt.show()
Part 7 · The Agentic AI Connection
Why this matters for AI Agents

Every agent you'll ever build uses supervised learning somewhere

When an AI agent decides which tool to call, it scores each option using a classifier. When it predicts whether a user query needs a web search or can be answered from memory, that's classification. When it estimates how long a task will take, that's regression. The Random Forest you just trained and the Logistic Regression you just ran — those are the same algorithms powering decision layers in production agents right now.

More specifically: RLHF (Reinforcement Learning from Human Feedback — the technique used to train Claude, ChatGPT, and every major LLM) has a supervised learning component at its core. A reward model is trained using labelled preference data (human A vs human B comparisons) — that's supervised classification. The reward model then guides the RL training. You can't understand RLHF without first understanding supervised learning.

RAG systems (Retrieval Augmented Generation — used in almost every production AI agent) use a reranking model to score document relevance. That reranker is a supervised regression model predicting "how relevant is this document to this query?" The embeddings it scores were trained with contrastive supervised learning.

The workflow every time

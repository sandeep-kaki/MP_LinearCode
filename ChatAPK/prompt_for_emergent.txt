SECTION 4 — END-TO-END ENCRYPTION IMPLEMENTATION
-------------------------------------------------------------

Key Generation:
- On first successful login, generate a TweetNaCl keypair 
  (nacl.box.keyPair())
- Store private key in Expo SecureStore (never sent anywhere)
- POST public key to Supabase users table via backend API
- On subsequent logins, load private key from SecureStore

Key Exchange:
- After login, fetch the OTHER user's public key from 
  Supabase users table
- Store it in Zustand state for use during encryption

Encrypting a Message:
- Generate a random nonce: nacl.randomBytes(nacl.box.nonceLength)
- Encrypt: nacl.box(message, nonce, recipientPublicKey, myPrivateKey)
- Store: base64-encoded ciphertext + base64-encoded nonce in DB
- NEVER store plaintext in DB

Decrypting a Message:
- On receive, decode ciphertext and nonce from base64
- Decrypt: nacl.box.open(ciphertext, nonce, senderPublicKey, myPrivateKey)
- Decryption happens entirely on device
- If decryption fails, show "Unable to decrypt message"

File Encryption:
- Read file as ArrayBuffer
- Encrypt raw bytes using same nacl.box method
- Upload encrypted bytes to Supabase Storage
- Store the signed URL + nonce in the files table in DB

SECTION 5 — SUPABASE DATABASE SCHEMA
-------------------------------------------------------------

Create these exact tables with these exact columns.
Also provide the complete SQL file I can run in Supabase 
SQL editor.

Table: users
- id: uuid, primary key, default gen_random_uuid()
- user_identity: text, unique, not null (values: 'user_a' or 'user_b')
- public_key: text, not null (base64 encoded nacl public key)
- created_at: timestamptz, default now()
- last_seen: timestamptz

Table: messages
- id: uuid, primary key, default gen_random_uuid()
- sender_identity: text, not null
- recipient_identity: text, not null
- encrypted_content: text, not null (base64 encoded ciphertext)
- nonce: text, not null (base64 encoded nonce)
- file_id: uuid, nullable, references files(id)
- is_delivered: boolean, default false
- created_at: timestamptz, default now()

Table: files
- id: uuid, primary key, default gen_random_uuid()
- sender_identity: text, not null
- recipient_identity: text, not null
- encrypted_url: text, not null (Supabase Storage signed URL)
- file_name: text, not null
- file_type: text, not null (image/pdf/video/doc)
- file_size_bytes: bigint
- nonce: text, not null
- created_at: timestamptz, default now()

Table: call_signals
- id: uuid, primary key, default gen_random_uuid()
- caller_identity: text, not null
- callee_identity: text, not null
- room_id: text, not null
- status: text, not null (values: 'ringing', 'accepted', 
  'rejected', 'ended')
- call_type: text, not null (values: 'video', 'voice')
- created_at: timestamptz, default now()

Row Level Security Rules:
- users table: a user can only read/update their own row, 
  but CAN read the other user's public_key column only
- messages table: a user can only read rows where their 
  identity is sender OR recipient
- files table: same as messages
- call_signals table: same as messages
- All RLS must use the JWT user_identity claim for checks
- Include all RLS SQL in the migration file

Supabase Storage:
- Create a private bucket called "encrypted-files"
- Only authenticated users (valid JWT) can upload/download
- Signed URLs expire after 2 hours

SECTION 6 — BACKEND API (NODE.JS ON RAILWAY)
-------------------------------------------------------------

Endpoints:

POST /auth/verify-code
- Body: { invite_code: string }
- Verify invite code against bcrypt hashed env vars
- On success: create or fetch user record in Supabase, 
  return signed JWT ({ user_identity, exp: 30 days })
- On fail: return 401

POST /auth/register-public-key
- Protected route (requires valid JWT in Authorization header)
- Body: { public_key: string }
- Upsert public key into users table in Supabase

GET /auth/get-peer-key
- Protected route
- Returns the OTHER user's public key from Supabase
- If user_identity is user_a, return user_b's key and vice versa

POST /calls/create-token
- Protected route
- Body: { room_id: string, call_type: string }
- Generate 100ms room token using 100ms Management API
- Return the room token to the app
- Use 100ms API credentials from env vars

GET /health
- Returns { status: "ok", timestamp: Date.now() }
- No auth required

Environment Variables needed (document all of these):
- SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY
- JWT_SECRET (random 64 char string)
- USER_A_CODE_HASH (bcrypt hash of "PHANTOM-ALPHA-001")
- USER_B_CODE_HASH (bcrypt hash of "PHANTOM-BETA-002")
- HMS_APP_ACCESS_KEY (from 100ms dashboard)
- HMS_APP_SECRET (from 100ms dashboard)
- HMS_TEMPLATE_ID (from 100ms dashboard)
- PORT (Railway sets this automatically)
